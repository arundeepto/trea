Firebase Setup Guide

Complete these one-time setup steps to connect your Firebase project:

  1. Go to the Firebase Console and create a project.
  2. Build → Authentication → Sign-in method → enable Email/Password.
  3. Build → Firestore Database → create a database (production mode).
  4. Paste the following into your Firestore rules: 
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    function isSignedIn() { return request.auth != null; }
    function myRole() {
      return get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role;
    }
    function isTreasurer() {
      return isSignedIn() &&
        (!exists(/databases/$(database)/documents/users/$(request.auth.uid)) || myRole() == 'treasurer');
    }
    function isGirlsAdmin() {
      return isSignedIn() && exists(/databases/$(database)/documents/users/$(request.auth.uid)) && myRole() == 'girls_admin';
    }
    function isFemaleRoll(roll) {
      return exists(/databases/$(database)/documents/students/$(string(roll))) &&
        get(/databases/$(database)/documents/students/$(string(roll))).data.gender == 'female';
    }

    // A new sign-up may create their OWN profile doc, but only with role "pending" — nobody
    // can hand themselves a role from the browser. Afterwards they may update harmless fields
    // (like their name) but can never change their own role/roll. Only the treasurer, or you
    // manually from the Firebase Console (which isn't subject to these rules), can promote
    // someone to treasurer / subscriber / girls_admin.
    match /users/{uid} {
      allow read: if isSignedIn();
      allow create: if isSignedIn() && request.auth.uid == uid
                    && request.resource.data.role == 'pending';
      allow update: if isSignedIn() && request.auth.uid == uid
                    && request.resource.data.role == resource.data.role
                    && request.resource.data.roll == resource.data.roll;
      allow write: if isTreasurer();
    }

    // a subscriber may create sheet-print orders only under their own uid;
    // the treasurer can edit/delete anything; the girls' admin may only update
    // (to mark "received") — not delete
    match /sheetOrders/{orderId} {
      allow read: if isSignedIn();
      allow create: if isSignedIn() && request.resource.data.orderedByUid == request.auth.uid;
      allow update: if isTreasurer() || isGirlsAdmin();
      allow delete: if isTreasurer();
    }

    // the girls' admin may add a payment ONLY for a subscriber whose gender is 'female';
    // the treasurer may add a payment for anyone
    match /payments/{paymentId} {
      allow read: if isSignedIn();
      allow create: if isTreasurer() ||
        (isGirlsAdmin() && isFemaleRoll(request.resource.data.roll));
      allow update, delete: if isTreasurer();
    }

    // activity log entries — treasurer always, girls' admin only for her own actions
    match /activity/{activityId} {
      allow read: if isSignedIn();
      allow create: if isTreasurer() || isGirlsAdmin();
      allow update, delete: if isTreasurer();
    }

    match /{document=**} {
      allow read: if isSignedIn();
      allow write: if isTreasurer();
    }
  }
}
  5. Project settings → General → scroll down to "Your apps" → add a Web app and copy the firebaseConfig object.
  6. In this file, paste your own config in place of firebaseConfig at the start of the <script type="module"> block, then save/upload the file again.

These steps only need to be done once. After that, anyone can register with just a name, email, and password. New accounts start as "Pending" and see a waiting screen. To activate someone: open Firestore → users collection → find their document (match by email) → set role to treasurer, subscriber, or girls_admin. For a subscriber, also add a roll field (number) matching their entry in the students collection.
